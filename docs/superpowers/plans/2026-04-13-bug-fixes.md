### Task 1: Fix accent-insensitive item name lookup

**Root cause:** `normaliseItemName("Banner of Châlons (Standard bearer)")` produces `"banner of châlons"` (â preserved). `MAGIC_ITEM_MAP` is keyed by `item.name.toLowerCase()` from magic-items.js which uses `"Banner of Chalons"` (no accent) → key is `"banner of chalons"`. The lookup returns `undefined`; the unit gets `magicItems = []`; only the character's banner appears.

**Files:**

- Modify: `src/parsers/resolve.js:95-111`
- Test: `src/test/combat-banner.test.js`

- [ ] **Step 3: Fix normaliseItemName and buildMagicItemMap in resolve.js**

In `src/parsers/resolve.js`, replace `normaliseItemName` (lines 95–100):

```js
/**
 * Normalize an item name for lookup in MAGIC_ITEM_MAP
 */
export function normaliseItemName(name) {
  return name
    .replace(/\s*\(.*$/, "")
    .toLowerCase()
    .normalize("NFD")
    .replace(/[\u0300-\u036f]/g, "")
    .replace(/\*$/, "");
}
```

Replace `buildMagicItemMap` (lines 105–111):

```js
export function buildMagicItemMap() {
  const map = {};
  for (const item of MAGIC_ITEMS) {
    const key = item.name
      .toLowerCase()
      .normalize("NFD")
      .replace(/[\u0300-\u036f]/g, "");
    map[key] = item;
  }
  return map;
}
```

### Task 2: Hide Arch-Lightning Rod on opponent's turn

**Root cause:** `renderMagicItemsContext` is called on the opponent turn with `subPhaseId = null`. The guard `!subPhaseId && item.yourTurnOnly` only skips items with `yourTurnOnly: true`. Arch-Lightning Rod lacks that flag.

**Files:**

- Modify: `src/data/magic-items.js:463-471`
- Test: `src/test/screens/opponent-turn.test.js`

- [ ] **Step 1: Write the failing test**

```js
it("does not show Arch-Lightning Rod during opponent strategy phase", () => {
  const lizardArmy = loadArmy("lizardmen");
  startGame(lizardArmy);
  saveIsOpponentTurn(true);
  savePhaseIndex(0); // strategy phase
  renderOpponentTurnScreen(lizardArmy);
  expect(getApp().textContent).not.toContain("Arch-Lightning Rod");
});
```

All imports (`loadArmy`, `startGame`, `getApp`, `saveIsOpponentTurn`, `savePhaseIndex`, `renderOpponentTurnScreen`) are already present in the file.
